# Before you start

- Create a project API key in the Rewrite dashboard.
- Keep the key server-side only.
- Use a Brazilian destination number in E.164 format, for example `+5511999999999`.

## Send your first SMS

```bash
curl -X POST "https://api.rewritetoday.com/v1/messages" \
  -H "Authorization: Bearer rw_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: msg:quickstart:2026-03-19T18:30" \
  -d '{
    "to": "+5511999999999",
    "content": "Rewrite: your verification code is 123456"
  }'
```

Successful response:

```json
{
  "ok": true,
  "data": {
    "id": "748395130237498500",
    "createdAt": "2026-03-19T18:30:00.000Z",
    "analysis": {
      "characters": 39,
      "encoding": "gsm7",
      "segments": {
        "concat": 153,
        "count": 1,
        "reason": "fits",
        "single": 160
      }
    }
  }
}
```

## Common next steps

- Read the created resource later with `GET /messages/{id}`.
- Add [webhooks](/en/webhooks/introduction) to receive [`message.sent`](/en/webhooks/events/message-sent), [`message.failed`](/en/webhooks/events/message-failed), [`message.scheduled`](/en/webhooks/events/message-scheduled), [`message.canceled`](/en/webhooks/events/message-canceled), [`message.batch`](/en/webhooks/events/message-batch), and [`sms.otp`](/en/webhooks/events/sms-otp).
- Use [templates](/en/api/i18n) when you need locale-specific content.

## Troubleshooting

- `400 INVALID_JSON_BODY`: request body, phone number, or route-specific validation failed.
- `401 INVALID_OR_MISSING_API_KEY`: bearer token is missing or invalid.
- `403 PROJECT_SUBSCRIPTION_REQUIRED` or `PROJECT_BILLING_PAYMENT_REQUIRED`: project is blocked from sending.
- `429 RATE_LIMITED`: back off and honor the retry headers.
