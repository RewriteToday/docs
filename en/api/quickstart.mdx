export function SmsRequestPlayground() {
  const curlCommand = [
    'curl -X POST "https://api.rewritetoday.com/v1/messages" \\',
    '  -H "Authorization: Bearer (Your API Key here)" \\',
    '  -H "Content-Type: application/json" \\',
    "  -d '{",
    '    "to": "+15551234567",',
    '    "message": "Your OTP code is 123456",',
    '    "metadata": { "source": "quickstart" }',

}

## Before you start

- Create a Rewrite API key in your dashboard.
- Keep your API key server-side only.
- Use destination numbers in **E.164** format (example: `+15551234567`).

## Send your first SMS

<Steps>
  <Step title="Build your request" titleSize="h3">
    Use this base cURL request as your starting point.

    <SmsRequestPlayground />
  </Step>

  <Step title="Check the response" titleSize="h3">
    Run this request, then compare your output with the response example below:

    ```bash
    curl -X POST "https://api.rewritetoday.com/v1/messages" \
      -H "Authorization: Bearer rw_..." \
      -H "Content-Type: application/json" \
      -d '{
        "to": "+15551234567",
        "message": "Your OTP code is 123456",
        "metadata": { "source": "quickstart" }
      }'
    ```

    A successful request returns a created SMS object similar to:

    ```json
    {
      "id": "733501234567890123",
      "to": "+15551234567",
      "message": "Your OTP code is 123456",
      "status": "QUEUED"
    }
    ```
  </Step>
</Steps>

<Info>
Use your own verified/test number in development. Keep API keys and secrets out of client-side code.
</Info>

## Troubleshooting

<CardGroup cols={2}>
  <Card title="401 Unauthorized" icon="shield-alert" color="#C9CBD1">
    Your API key is missing or invalid. Confirm the bearer token in your cURL `Authorization` header.
  </Card>

  <Card title="422 Unprocessable Entity" icon="triangle-alert" color="#C9CBD1">
    Your payload is invalid. Check the `to` number format and required fields.
  </Card>
