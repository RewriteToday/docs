# Overview

Rewrite list endpoints use **cursor-based pagination**.

Every paginated response returns:

- `data`: the current page items
- `cursor`: pagination state for the next or previous request

## Response format

```json
{
  "ok": true,
  "data": [],
  "cursor": {
    "persist": true,
    "next": "748395130237498399",
    "prev": "748395130237498420"
  }
}
```

## Cursor fields

`persist`: Whether more results exist.<br/>
`next`: Cursor to use for the next page.<br/>
`prev`: Cursor to use for the previous page.

- Treat `cursor.next` and `cursor.prev` as opaque values.

## Query parameters

- `after`: Fetch results after a given ID.
- `before`: Fetch results before a given ID.
- `limit`: Number of items to return (2 to 100).

## Forward pagination

Use `after` to move forward.

<CodeGroup>
    ```bash CLI
    rewrite webhooks list --limit 25 --after 9213923891293040
    ```

    ```ts Node.js
    const rewrite = new Rewrite(process.env.REWRITE_API_KEY);

    const { data, error, cursor } = await rewrite.webhooks.list({
      limit: 25,
    });

    if (error) throw error;

    if (cursor?.persist) {
      const { data: nextData, error: nextError, cursor: nextCursor } =
        await rewrite.webhooks.list({
          limit: 25,
          after: cursor.next,
        });

      if (nextError) throw nextError;
    }
    ```

    ```go Golang
    package main

    import (
        rewrite "github.com/rewritetoday/golang"
    )

    func main() {
        client := rewrite.New("rw_abcxyz")

        data, cursor, err := client.Webhooks.List(25)
        if err != nil {
            panic(err)
        }

        if cursor != nil && cursor.Persist {
            nextData, nextCursor, err := client.Webhooks.List(
                25,
                rewrite.Webhooks.ListOptions{After: cursor.Next},
            )
            if err != nil {
                panic(err)
            }

            _, _ = nextData, nextCursor
        }

        _ = data
    }
    ```

    ```py Python
    import rewrite

    client = rewrite.Rewrite("rw_abcxyz")

    data, error, cursor = client.webhooks.list(limit=25)

    if error:
        raise Exception(error)

    if cursor and cursor.persist:
        next_data, next_error, next_cursor = client.webhooks.list(
            limit=25,
            after=cursor.next,
        )

        if next_error:
            raise Exception(next_error)
    ```

    ```bash cURL
    curl -X GET 'https://api.rewritetoday.com/v1/webhooks?limit=25&after=9213923891293040' \
         -H 'Authorization: Bearer rw_abcxyz'
    ```

    ```bash HTTPie
    http GET 'https://api.rewritetoday.com/v1/webhooks?limit=25&after=9213923891293040' \
         'Authorization: Bearer rw_abcxyz'
    ```
</CodeGroup>

## Common mistakes

<Tabs sync={false}>
  <Tab title="Using before and after">
      Do not send `before` and `after` in the same request.
  </Tab>
  <Tab title="Inferring the end from length">
      Do not use `length < limit` to detect the end of the list. Use `cursor.persist`.
  </Tab>
  <Tab title="Treating cursors as generated values">
      Do not generate cursors client-side. Reuse the values returned by the API.
  </Tab>
</Tabs>
