# Overview

Rewrite API v1 exposes project-level endpoints for sending SMS, verifying OTPs, managing templates and webhooks, and reading delivery logs.

## Base URL

All public endpoints live under:

```text
https://api.rewritetoday.com/v1
```

## Authentication

Public routes use your project API key:

```http
Authorization: Bearer rw_...
```

## Request conventions

- Use `Content-Type: application/json` on non-`GET` requests.
- Public send and OTP flows currently accept only Brazilian destination numbers starting with `+55`.
- Message creation uses `content` for inline SMS bodies.
- Template-backed sends use `templateId` plus `variables`.
- Create/send routes support `Idempotency-Key`.

Example public request:

```bash
curl -X POST "https://api.rewritetoday.com/v1/messages" \
  -H "Authorization: Bearer rw_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: msg:login-otp:2026-03-19T18:30" \
  -d '{
    "to": "+5511999999999",
    "content": "Rewrite: your verification code is 123456"
  }'
```

## Response conventions

Successful responses use the `ok` / `data` envelope:

```json
{
  "ok": true,
  "data": {
    "id": "748395130237498500",
    "createdAt": "2026-03-19T18:30:00.000Z",
    "analysis": {
      "characters": 37,
      "encoding": "gsm7",
      "segments": {
        "concat": 153,
        "count": 1,
        "reason": "fits",
        "single": 160
      }
    }
  }
}
```

Paginated list routes return top-level `cursor`.
