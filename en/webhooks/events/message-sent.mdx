---
title: "message.sent"
description: "SMS accepted by the provider after dispatch."
---

`message.sent` is emitted when Rewrite receives provider acceptance for an SMS dispatch.

```json
{
  "id": "748395130237498911",
  "createdAt": "2026-03-19T18:42:13.000Z",
  "type": "message.sent",
  "data": {
    "id": "748395130237498500",
    "projectId": "748395130237498412",
    "to": "+5511999999999",
    "tags": [{ "name": "use_case", "value": "otp_login" }],
    "type": "SMS",
    "status": "SENT",
    "content": "Rewrite: your verification code is 123456",
    "country": "br",
    "analysis": {
      "characters": 41,
      "encoding": "gsm7",
      "segments": { "concat": 153, "count": 1, "reason": "fits", "single": 160 }
    },
    "error": null,
    "deliveredAt": null,
    "scheduledAt": null,
    "templateId": null
  }
}
```

Notes:

- `data.status` is `SENT`.
- `data.error` is `null`.
- Immediate sends usually emit [`message.queued`](./message-queued) first.
- Messages previously accepted with `scheduledAt` also emit `message.sent` later and can keep their original `data.scheduledAt`.
