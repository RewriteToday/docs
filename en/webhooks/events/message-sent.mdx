---
title: "message.sent"
description: "Message accepted for provider delivery."
---

`message.sent` is emitted when Rewrite receives provider acceptance for a message dispatch.

```json
{
  "id": "748395130237498911",
  "createdAt": "2026-03-19T18:42:13.000Z",
  "type": "message.sent",
  "sandbox": false,
  "data": {
    "id": "748395130237498500",
    "projectId": "748395130237498412",
    "contact": "Fernanda",
    "contactId": "748395130237498515",
    "to": "+5511999999999",
    "tags": { "flow": "otp_login", "origin": "checkout" },
    "sandbox": false,
    "type": "SMS",
    "status": "SENT",
    "content": "Rewrite: your verification code is 123456",
    "country": "br",
    "analysis": {
      "characters": 41,
      "encoding": "gsm7",
      "segments": { "concat": 153, "count": 1, "reason": "fits", "single": 160 }
    },
    "error": null,
    "deliveredAt": null,
    "scheduledAt": null,
    "templateId": null
  }
}
```

Notes:

- `data.status` is `SENT`.
- `data.error` is `null`.
- Immediate sends usually emit [`message.queued`](./message-queued) before this event.
- Scheduled sends can emit `message.sent` later while preserving the original `data.scheduledAt`.
