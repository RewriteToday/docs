---
title: "message.queued"
description: "Immediate SMS persisted in queue before provider dispatch."
---

`message.queued` is emitted as soon as Rewrite persists an immediate SMS for dispatch.

```json
{
  "id": "748395130237498913",
  "createdAt": "2026-03-19T18:42:11.000Z",
  "type": "message.queued",
  "data": {
    "id": "748395130237498500",
    "projectId": "748395130237498412",
    "to": "+5511999999999",
    "tags": [{ "name": "use_case", "value": "otp_login" }],
    "type": "SMS",
    "status": "QUEUED",
    "content": "Rewrite: your verification code is 123456",
    "country": "br",
    "analysis": {
      "characters": 39,
      "encoding": "gsm7",
      "segments": { "concat": 153, "count": 1, "reason": "fits", "single": 160 }
    },
    "error": null,
    "deliveredAt": null,
    "scheduledAt": null,
    "templateId": null
  }
}
```

Notes:

- `message.queued` is used for immediate sends.
- Scheduled sends use [`message.scheduled`](./message-scheduled) instead.
