Rewrite themes bring the Rewrite visual language into your editor.

The official `RewriteToday/theme` repository ships matching dark and light variants for Zed, Visual Studio Code, and Neovim.

Repository: [github.com/RewriteToday/theme](https://github.com/RewriteToday/theme)

## Theme variants

<CardGroup cols={2}>
  <Card title="Rewrite Night" icon="moon" color="#C9CBD1">
    Dark mode inspired by Rewrite landing page code previews and the darker product atmosphere.
  </Card>

  <Card title="Rewrite Day" icon="sun" color="#C9CBD1">
    Light mode with the same hierarchy and amber accent in a warmer editor palette.
  </Card>

  <Card title="Shared visual language" icon="palette" color="#C9CBD1">
    Keep the same color relationships, contrast decisions, and accent logic across every supported editor.
  </Card>

  <Card title="Editor-specific packaging" icon="box" color="#C9CBD1">
    Each editor keeps its own install flow while preserving one Rewrite look and feel.
  </Card>
</CardGroup>

## Supported editors

<CardGroup cols={3}>
  <Card title="Zed" icon="bolt" href="./zed" color="#C9CBD1">
    Install directly from the Zed extensions marketplace and choose Night or Day.
  </Card>

  <Card title="VS Code" icon="code" href="./vscode" color="#C9CBD1">
    Install from the Visual Studio Code Marketplace with the `rewritetoday.rewrite` extension.
  </Card>

  <Card title="Neovim" icon="terminal" href="./neovim" color="#C9CBD1">
    Install from GitHub and choose `rewrite`, `rewrite-night`, or `rewrite-day`.
  </Card>
</CardGroup>

## When should I use Themes?

<Accordion title="I want Rewrite colors inside my editor">
  Start with Themes when you want the same Rewrite palette from marketing and product surfaces in your daily coding environment.
</Accordion>

<Accordion title="I want matching dark and light variants">
  Use Themes if you switch between dark and light mode and want the two variants to stay visually aligned across editors.
</Accordion>

<Accordion title="I want editor-native installation flows">
  Each supported editor uses the install path that feels natural for that platform: marketplace for Zed and VS Code, plugin-manager setup for Neovim.
</Accordion>
