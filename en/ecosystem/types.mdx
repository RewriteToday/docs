`@rewritetoday/types` centralizes Rewrite API contracts so your integrations stay consistent across services and packages.

Repository: [github.com/RewriteToday/types](https://github.com/RewriteToday/types)

## What is included

<CardGroup cols={2}>
  <Card title="Routes" icon="signs-post" color="#C9CBD1">
    Route builders such as `Routes.apiKeys.create()`.
  </Card>
  <Card title="Enums" icon="list" color="#C9CBD1">
    Shared enums for scopes, event types, statuses, and domain constants.
  </Card>
  <Card title="Model contracts" icon="brackets-curly" color="#C9CBD1">
    Request and response interfaces used across Rewrite SDKs and tooling.
  </Card>
  <Card title="Utility types" icon="wrench" color="#C9CBD1">
    Reusable helper types for stronger compile-time guarantees.
  </Card>
</CardGroup>

## Route builder example

```ts
import { Routes } from '@rewritetoday/types';

const listRoute = Routes.apiKeys.list();
const createRoute = Routes.apiKeys.create();

console.log({ listRoute, createRoute });
```

## Enum example

```ts
import { APIKeyScope, WebhookEventType } from '@rewritetoday/types';

const scopes = [APIKeyScope.ReadTemplates, APIKeyScope.ReadWebhooks];
const events = [WebhookEventType.SMSDelivered, WebhookEventType.SMSFailed];

console.log({ scopes, events });
```

## Why use it even without REST?

- Keeps string literals out of business code.
- Improves refactoring safety in large codebases.
- Makes route and contract drift visible during code review.
- Standardizes API usage across teams and services.

<Info>
Using `@rewritetoday/types` with `@rewritetoday/rest` keeps request paths, enums, and payload contracts aligned end-to-end.
</Info>
