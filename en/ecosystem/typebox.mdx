[`@rewritetoday/typebox`](https://www.npmjs.com/package/@rewritetoday/typebox) provides official TypeBox schemas for the Rewrite API so you can validate requests, responses, reusable resources, and webhook payloads at runtime.

Repository: [github.com/RewriteToday/typebox](https://github.com/RewriteToday/typebox)

## When to use it

- You prefer TypeBox as your runtime validation layer.
- You want runtime validation in addition to TypeScript types.
- You receive external input before calling Rewrite endpoints.
- You need versioned schemas that stay aligned with Rewrite API models.

## Request validation example

```ts
import { RESTPostCreateTemplateBody } from '@rewritetoday/typebox/v1';
import { Parse } from '@sinclair/typebox/value';

const data = Parse(RESTPostCreateTemplateBody, {
  name: 'order_shipped',
  content: 'Hi {{first_name}}, your order {{order_id}} is on the way.',
  variables: [
    { name: 'first_name', fallback: 'customer' },
    { name: 'order_id', fallback: '0000' },
  ],
});

console.log(data);
```

## Webhook validation example

```ts
import { WebhookEvent } from '@rewritetoday/typebox/v1';
import { Parse } from '@sinclair/typebox/value';

let event;

try {
  event = Parse(WebhookEvent, payload);
} catch (error) {
  console.error({ error });
  throw new Error('Invalid Rewrite webhook payload');
}

console.log({ data: event });
```

## Best practices

<Accordion title="Keep versions explicit">
  Import from a specific API namespace such as `@rewritetoday/typebox/v1` so upgrades stay deliberate.
</Accordion>

<Accordion title="Validate at the edges">
  Parse external input before it reaches your business logic so invalid payloads fail fast and predictably.
</Accordion>

<Accordion title="Pair runtime and transport layers">
  Combine `@rewritetoday/typebox` with `@rewritetoday/rest` and `@rewritetoday/types` for validation, routing, and transport in one stack.
</Accordion>
