`@rewritetoday/zod` provides official Zod schemas for the Rewrite API so you can validate requests, responses, reusable resources, and webhook payloads at runtime.

Repository: [github.com/RewriteToday/zod](https://github.com/RewriteToday/zod)

## When to use it

- You want runtime validation in addition to TypeScript types.
- You receive external input before calling Rewrite endpoints.
- You need to validate webhook payloads before handling them.
- You want versioned schemas that stay aligned with Rewrite API models.

## Request validation example

```ts
import { RESTPostCreateTemplateBody } from '@rewritetoday/zod/v1';

const data = RESTPostCreateTemplateBody.parse({
  name: 'order_shipped',
  content: 'Hi {{first_name}}, your order {{order_id}} is on the way.',
  variables: [
    { name: 'first_name', fallback: 'customer' },
    { name: 'order_id', fallback: '0000' },
  ],
});

console.log(data);
```

## Webhook validation example

```ts
import { WebhookEvent } from '@rewritetoday/zod/v1';

const result = WebhookEvent.safeParse(payload);

if (!result.success) {
  console.error({ error: result.error.flatten() });
  throw new Error('Invalid Rewrite webhook payload');
}

console.log({ data: result.data });
```

## Best practices

<Accordion title="Validate at the edges">
  Parse external input before it reaches your business logic so invalid payloads fail fast and predictably.
</Accordion>

<Accordion title="Keep versions explicit">
  Import from a specific API namespace such as `@rewritetoday/zod/v1` so upgrades stay deliberate.
</Accordion>

<Accordion title="Pair runtime and transport layers">
  Combine `@rewritetoday/zod` with `@rewritetoday/rest` and `@rewritetoday/types` for validation, routing, and transport in one stack.
</Accordion>
