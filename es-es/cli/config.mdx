## Formatos de salida

La **CLI** admite dos formatos de salida configurables mediante `--output` o `-o`.

### Texto (predeterminado)

Salida legible para humanos, optimizada para uso interactivo.

```bash
rewrite whoami
```

```
You are logged in project almeida's project
```

### JSON

Salida estructurada para scripts, pipelines y CI/CD.

```bash
rewrite whoami -o json
```

```json
{
  "project": "John's Blog",
  "api_key": "rw_...",
  "status": "AUTHENTICATED"
}
```

#### Úsalo con jq para filtrar

```bash
rewrite -o json logs list | jq '.data[] | select(.status=="failed")'
```

## Autenticación y almacenamiento de tokens

Tu token de API se almacena de forma segura en el keyring nativo del sistema.

- macOS: Keychain
- Windows: Credential Manager
- Linux: gnome-keyring o kwallet

<Accordion title="Linux: ¿el token no se guarda?">

Instala un backend de keyring.

```bash
# Debian / Ubuntu
sudo apt install gnome-keyring
```

```bash
# Fedora
sudo dnf install gnome-keyring
```

Luego vuelve a intentar:

```bash
rewrite login
```

</Accordion>

## Comportamiento de color y CI

- Los colores se desactivan automáticamente cuando:
  - se usa el flag `--no-color`.
  - `NO_COLOR=true` está en las variables de entorno.
  - la salida no es un **TTY**.
- Cuando `CI=true`, los prompts interactivos se desactivan automáticamente.
