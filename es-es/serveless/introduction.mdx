Serverless es una de las formas más rápidas de integrar Rewrite en producción.

Usa el SDK oficial para enviar SMS desde plataformas como **Vercel Functions**, **Cloudflare Workers** y **Supabase Edge Functions** sin gestionar servidores.

## ¿Por qué serverless?

## Patrón común

Todos los runtimes serverless siguen el mismo flujo:

1. Leer secretos desde variables de entorno
2. Instanciar `Rewrite`
3. Validar el payload de la solicitud
4. Llamar a `rewrite.messages.send(...)`
5. Devolver una respuesta JSON con el código de estado correcto

<Info>
Mantén tu API key solo en el servidor. Nunca expongas `REWRITE_API_KEY` en bundles del cliente.
</Info>
