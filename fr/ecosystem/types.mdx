`@rewritejs/types` centralise les contrats API Rewrite pour garder de la coherence entre services et packages.

Depot : [github.com/RewriteToday/types](https://github.com/RewriteToday/types)

## Exemple route builders

```ts
import { Routes } from '@rewritejs/types';

const listRoute = Routes.apiKeys.list('748395130237498368');
const createRoute = Routes.apiKeys.create('748395130237498368');

console.log({ listRoute, createRoute });
```

## Exemple enums

```ts
import { APIKeyScope, WebhookEventType } from '@rewritejs/types';

const scopes = [APIKeyScope.ReadTemplates, APIKeyScope.ReadWebhooks];
const events = [WebhookEventType.SMSDelivered, WebhookEventType.SMSFailed];

console.log({ scopes, events });
```

## Pourquoi l'utiliser meme sans REST ?

- Evite les string literals disperses dans le code metier.
- Ameliore la securite de refactor sur des codebases volumineuses.
- Rend visible la derive routes/contrats pendant la review.
- Standardise l'usage de l'API entre equipes et services.

<Info>
Utiliser `@rewritejs/types` avec `@rewritejs/rest` maintient routes, enums et payloads alignes de bout en bout.
</Info>
