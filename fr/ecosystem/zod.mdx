`@rewritejs/zod` fournit des schemas Zod officiels pour l'API Rewrite afin de valider requests, responses, ressources reutilisables et payloads de webhook en runtime.

Depot : [github.com/RewriteToday/zod](https://github.com/RewriteToday/zod)

## Quand l'utiliser

- Quand vous voulez une validation runtime en plus des types TypeScript.
- Quand vous recevez des inputs externes avant d'appeler les endpoints Rewrite.
- Quand vous devez valider des payloads de webhook avant traitement.
- Quand vous voulez des schemas versionnes alignes sur les modeles de l'API Rewrite.

## Exemple de validation de request

```ts
import { RESTPostCreateTemplateBody } from '@rewritejs/zod/v1';

const data = RESTPostCreateTemplateBody.parse({
  name: 'order_shipped',
  content: 'Hi {{first_name}}, your order {{order_id}} is on the way.',
  variables: [
    { name: 'first_name', fallback: 'customer' },
    { name: 'order_id', fallback: '0000' },
  ],
});

console.log(data);
```

## Exemple de validation de webhook

```ts
import { WebhookEvent } from '@rewritejs/zod/v1';

const result = WebhookEvent.safeParse(payload);

if (!result.success) {
  console.error({ error: result.error.flatten() });
  throw new Error('Payload de webhook Rewrite invalide');
}

console.log({ data: result.data });
```

## Bonnes pratiques

<Accordion title="Validez aux frontieres">
  Parsez les inputs externes avant qu'ils atteignent votre logique metier pour echouer tot et de maniere previsible.
</Accordion>

<Accordion title="Gardez des versions explicites">
  Importez depuis un namespace precis comme `@rewritejs/zod/v1` afin de garder des upgrades volontaires.
</Accordion>

<Accordion title="Combinez validation et transport">
  Utilisez `@rewritejs/zod` avec `@rewritejs/rest` et `@rewritejs/types` pour couvrir validation, routes et transport dans une seule stack.
</Accordion>
