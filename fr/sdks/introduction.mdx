## Que sont les Rewrite SDKs ?

Les **Rewrite SDKs** (Software Development Kits) sont des bibliothèques officielles qui simplifient la communication avec la Rewrite API.  
Ils fournissent des interfaces ergonomiques propres à chaque langage pour vous permettre d’intégrer des SMS transactionnels dans votre application rapidement et en toute sécurité, sans gérer les détails HTTP de bas niveau.

Les SDKs sont conçus pour être naturels dans chaque écosystème tout en restant entièrement alignés avec l’API sous-jacente.

<Card title="Pourquoi utiliser les Rewrite SDKs ?" horizontal>
  * **Intégration simplifiée** : Méthodes prêtes à l’emploi pour tous les endpoints principaux de l’API
  * **Typage fort** : Support des types de premier plan lorsqu’il est disponible
  * **Gestion des erreurs cohérente** : Modèles d’erreur unifiés entre les langages
  * **Courbe d’apprentissage réduite** : Pas besoin de comprendre chaque détail de l’API au départ
  * **Synchronisés** : Les SDKs sont mis à jour en même temps que les nouvelles fonctionnalités de l’API
</Card>

## Questions fréquentes

<Accordion title="Comment installer le SDK pour mon langage ?">
  Chaque SDK inclut des instructions d’installation dans son dépôt GitHub. Dans la plupart des cas, vous pouvez l’installer avec le gestionnaire de paquets de votre langage, comme npm, pip, go modules, Composer ou Bundler.
</Accordion>

<Accordion title="Les SDKs fonctionnent-ils en développement et en production ?">
  Oui. Tous les SDKs fonctionnent dans les deux environnements. Sans magie, sans boilerplate.
</Accordion>

<Accordion title="Que faire si je trouve un bug dans un SDK ?">
  Ouvrez une issue dans le dépôt GitHub du SDK concerné. Nous surveillons activement les issues et publions régulièrement des correctifs.
</Accordion>

<Accordion title="Il n’existe pas de SDK pour mon langage. Que faire ?">
  Indiquez-nous le langage dont vous avez besoin. Les contributions de la communauté sont les bienvenues, et nous sommes heureux de prendre en charge de nouveaux SDKs officiels lorsque c’est possible.
</Accordion>
