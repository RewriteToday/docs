## Formats de sortie

La **CLI** prend en charge deux formats de sortie configurables via `--output` ou `-o`.

### Texte (par défaut)

Sortie lisible par l’humain, optimisée pour une utilisation interactive.

```bash
rewrite whoami
```

```
You are logged in project almeida's project
```

### JSON

Sortie structurée pour les scripts, pipelines et CI/CD.

```bash
rewrite whoami -o json
```

```json
{
  "project": "John's Blog",
  "api_key": "rw_...",
  "status": "AUTHENTICATED"
}
```

#### Utilisation avec jq pour filtrer

```bash
rewrite -o json logs list | jq '.data[] | select(.status=="failed")'
```

## Authentification et stockage du token

Votre API token est stocké de manière sécurisée dans le keyring natif du système

- macOS : Keychain
- Windows : Credential Manager
- Linux : gnome-keyring ou kwallet

<Accordion title="Linux : le token n’est pas enregistré ?">

Installez un backend keyring

```bash
# Debian / Ubuntu
sudo apt install gnome-keyring
```

```bash
# Fedora
sudo dnf install gnome-keyring
```

Ensuite, réessayez :

```bash
rewrite login
```

</Accordion>

## Couleurs et comportement CI

- Les couleurs sont automatiquement désactivées lorsque :
  - le flag `--no-color` est utilisé.
  - `NO_COLOR=true` dans les variables d’environnement.
  - La sortie n’est pas un **TTY**.
- Quand `CI=true`, les prompts interactifs sont automatiquement désactivés.
