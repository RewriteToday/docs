Le serverless est l'une des manieres les plus rapides d'integrer Rewrite en production.

Utilisez le SDK officiel pour envoyer des SMS depuis des plateformes comme **Vercel Functions**, **Cloudflare Workers** et **Supabase Edge Functions**, sans gerer de serveurs.

## Pourquoi serverless ?

## Schema commun

Tous les runtimes serverless suivent le meme flux :

1. Lire les secrets depuis les variables d'environnement
2. Instancier `Rewrite`
3. Valider la charge utile de la requete
4. Appeler `rewrite.messages.send(...)`
5. Retourner une reponse JSON avec le code de statut adapte

<Info>
Conservez votre cle API uniquement cote serveur. N'exposez jamais `REWRITE_API_KEY` dans des bundles client.
</Info>
