## Formatos de saida

A **CLI** suporta dois formatos de saida configuraveis por `--output` ou `-o`.

### Texto (padrao)

Saida amigavel para uso interativo.

```bash
rewrite whoami
```

```
You are logged in project almeida's project
```

### JSON

Saida estruturada para scripts, pipelines e CI/CD.

```bash
rewrite whoami -o json
```

```json
{
  "project": "John's Blog",
  "api_key": "rw_...",
  "status": "AUTHENTICATED"
}
```

#### Use com jq para filtrar

```bash
rewrite -o json logs list | jq '.data[] | select(.status=="failed")'
```

## Autenticacao e armazenamento de token

Seu token de API e armazenado com seguranca no keyring nativo do sistema:

- macOS: Keychain
- Windows: Credential Manager
- Linux: gnome-keyring ou kwallet

<Accordion title="Linux: token nao esta salvando?">

Instale um backend de keyring:

```bash
# Debian / Ubuntu
sudo apt install gnome-keyring
```

```bash
# Fedora
sudo dnf install gnome-keyring
```

Depois tente novamente:

```bash
rewrite login
```

</Accordion>

## Cor e comportamento em CI

- As cores sao desativadas automaticamente quando:
  - a flag `--no-color` e usada.
  - `NO_COLOR=true` nas variaveis de ambiente.
  - a saida nao e um **TTY**.
- Quando `CI=true`, prompts interativos sao desativados automaticamente.
