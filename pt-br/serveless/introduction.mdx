Serverless e uma das formas mais rapidas de integrar a Rewrite em producao.

Use o SDK oficial para enviar SMS em plataformas como **Vercel Functions**, **Cloudflare Workers** e **Supabase Edge Functions** sem gerenciar servidores.

## Por que serverless?

<CardGroup cols={2}>
  <Card title="Deploy rapido" icon="rocket" color="#C9CBD1">
    Publique globalmente em minutos com setup minimo de infraestrutura.
  </Card>

  <Card title="Pague por execucao" icon="coins" color="#C9CBD1">
    Pague apenas pelas requisicoes que realmente processar.
  </Card>

  <Card title="Proximidade de edge" icon="globe" color="#C9CBD1">
    Reduza latencia executando perto dos seus usuarios.

  <Card title="Escala simples" icon="trending-up" color="#C9CBD1">

  </Card>

## Padrao comum

Todos os runtimes serverless seguem o mesmo fluxo:

1. Ler segredos de variaveis de ambiente/bindings
2. Instanciar `Rewrite`
3. Validar payload da requisicao
4. Chamar `rewrite.messages.send(...)`
5. Retornar JSON com status code correto

<Info>
Mantenha sua API key apenas no backend. Nunca exponha `REWRITE_API_KEY` no bundle do cliente.
</Info>
