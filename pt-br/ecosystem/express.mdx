[`@rewritetoday/express`](https://www.npmjs.com/package/@rewritetoday/express) e o adapter da Rewrite para apps Express que precisam de um endpoint de webhook nativo do framework com setup minimo.

Repositorio: [github.com/RewriteToday/adapters](https://github.com/RewriteToday/adapters)

## Quando usar

- Quando sua aplicacao ja usa roteamento e middleware do Express.
- Quando quer que o adapter cuide do raw body necessario para validar assinaturas.
- Quando prefere callbacks nomeados a parse manual de request.
- Quando precisa de uma rota de entrada dedicada e validada para webhooks da Rewrite.

## Exemplo basico

```ts
import express from 'express';
import { Webhooks } from '@rewritetoday/express';

const app = express();

app.post(
  '/webhooks/rewrite',
  ...Webhooks({
    secret: process.env.REWRITE_WEBHOOK_SECRET!,
    onPayload({ id, type, data }) {
      console.log({ id, type, data });
    },
  }),
);
```

## Boas praticas

<Accordion title="Use uma rota dedicada para o webhook">
  Mantenha o endpoint da Rewrite isolado de middlewares JSON genericos para que o tratamento de raw body continue previsivel.
</Accordion>

<Accordion title="Retorne 2xx rapidamente">
  Confirme a entrega assim que possivel e mova trabalho demorado para filas, jobs ou processadores em background.
</Accordion>

<Accordion title="Deduplique pelo id do evento">
  Armazene o id do webhook e torne efeitos colaterais idempotentes para que retries sejam seguros.
</Accordion>

<Accordion title="Callbacks de evento disponiveis">
  O adapter suporta `onSMSOTP`, `onMessageSent`, `onMessageBatch`, `onMessageQueued`, `onMessageDelivered`, `onMessageScheduled`, `onMessageFailed`, `onMessageCanceled` e o hook generico `onPayload`.
</Accordion>
