`@rewritetoday/types` centraliza os contratos da API da Rewrite para manter consistencia entre servicos e pacotes.

Repositorio: [github.com/RewriteToday/types](https://github.com/RewriteToday/types)

## O que inclui

## Exemplo com builders de rota

```ts
import { Routes } from '@rewritetoday/types';

const listRoute = Routes.apiKeys.list();
const createRoute = Routes.apiKeys.create();

console.log({ listRoute, createRoute });
```

## Exemplo com enums

```ts
import { APIKeyScope, WebhookEventType } from '@rewritetoday/types';

const scopes = [APIKeyScope.ReadTemplates, APIKeyScope.ReadWebhooks];
const events = [WebhookEventType.SMSDelivered, WebhookEventType.SMSFailed];

console.log({ scopes, events });
```

## Por que usar mesmo sem REST?

- Remove strings soltas do codigo de negocio.
- Aumenta seguranca de refatoracao em codebases grandes.
- Deixa divergencia de rota/contrato evidente no code review.
- Padroniza uso da API entre times e servicos.

<Info>
Usar `@rewritetoday/types` junto com `@rewritetoday/rest` mantem paths, enums e payloads alinhados de ponta a ponta.
</Info>
