[`@rewritetoday/fastify`](https://www.npmjs.com/package/@rewritetoday/fastify) e o adapter da Rewrite para servicos Fastify que preferem registrar webhooks no formato de plugin.

Repositorio: [github.com/RewriteToday/adapters](https://github.com/RewriteToday/adapters)

## Quando usar

- Quando seu servico ja usa Fastify e seu modelo de plugins.
- Quando quer que o registro do webhook respeite os limites de encapsulamento do Fastify.
- Quando prefere callbacks tipados a validacao de assinatura e parse manual do payload.
- Quando quer que o adapter registre o comportamento de parser necessario para validar webhooks.

## Exemplo basico

```ts
import Fastify from 'fastify';
import { Webhooks } from '@rewritetoday/fastify';

const app = Fastify();

await app.register(Webhooks, {
  path: '/webhooks/rewrite',
  secret: process.env.REWRITE_WEBHOOK_SECRET!,
  onPayload({ id, type, data }) {
    console.log({ id, type, data });
  },
});
```

## Boas praticas

<Accordion title="Mantenha a rota de webhook isolada">
  Deixe o adapter ser o dono da rota da Rewrite para que parsing e verificacao de assinatura continuem previsiveis.
</Accordion>

<Accordion title="Confirme a entrega rapido e processe de forma assincrona">
  Retorne sucesso cedo e entregue o trabalho lento para filas ou workers em background, reduzindo retries.
</Accordion>

<Accordion title="Torne os handlers idempotentes">
  Persista o id do evento e proteja escritas downstream contra entregas repetidas durante retries.
</Accordion>

<Accordion title="Callbacks de evento disponiveis">
  O adapter suporta `onSMSOTP`, `onMessageSent`, `onMessageBatch`, `onMessageQueued`, `onMessageDelivered`, `onMessageScheduled`, `onMessageFailed`, `onMessageCanceled` e o hook generico `onPayload`.
</Accordion>
