Os themes da Rewrite levam a linguagem visual da Rewrite para dentro do seu editor.

O repositorio oficial `RewriteToday/theme` distribui variantes dark e light alinhadas entre Zed, Visual Studio Code e Neovim.

Repositorio: [github.com/RewriteToday/theme](https://github.com/RewriteToday/theme)

## Variantes de tema

<CardGroup cols={2}>
  <Card title="Rewrite Night" icon="moon" color="#C9CBD1">
    Modo escuro inspirado nos code previews da landing page e na atmosfera mais escura da Rewrite.
  </Card>

  <Card title="Rewrite Day" icon="sun" color="#C9CBD1">

  <Card title="Linguagem visual compartilhada" icon="palette" color="#C9CBD1">

  <Card title="Empacotamento por editor" icon="box" color="#C9CBD1">

## Editores suportados

<CardGroup cols={3}>
  <Card title="Zed" icon="bolt" href="./zed" color="#C9CBD1">
    Instale direto do marketplace do Zed e escolha entre Night e Day.
  </Card>

  <Card title="VS Code" icon="code" href="./vscode" color="#C9CBD1">
    Instale pelo marketplace do Visual Studio Code com a extensao `rewritetoday.rewrite`.
  </Card>

  <Card title="Neovim" icon="terminal" href="./neovim" color="#C9CBD1">
    Instale via GitHub e escolha entre `rewrite`, `rewrite-night` ou `rewrite-day`.
  </Card>
</CardGroup>

## Quando devo usar Themes?

<Accordion title="Quero as cores da Rewrite dentro do meu editor">
  Comece por Themes quando quiser a mesma paleta da Rewrite das superficies de marketing e produto no seu ambiente diario de codigo.
</Accordion>

<Accordion title="Quero variantes dark e light alinhadas">
  Use Themes se voce alterna entre dark e light mode e quer manter as duas variantes visualmente coerentes entre editores.
</Accordion>

<Accordion title="Quero fluxos nativos de instalacao por editor">
  Cada editor usa o fluxo que faz sentido para a plataforma: marketplace para Zed e VS Code, setup via plugin manager para Neovim.
</Accordion>
