[`@rewritetoday/hono`](https://www.npmjs.com/package/@rewritetoday/hono) e o adapter da Rewrite para apps Hono rodando em edge ou em runtimes compativeis com Node.

Repositorio: [github.com/RewriteToday/adapters](https://github.com/RewriteToday/adapters)

## Quando usar

- Quando sua aplicacao ja expoe rotas HTTP com Hono.
- Quando quer um handler de webhook enxuto e alinhado ao modelo de request do Hono.
- Quando prefere callbacks tipados a codigo manual de verificacao de assinatura.
- Quando precisa de uma integracao que possa migrar entre edge e servidor com poucas mudancas.

## Capacidades principais

<CardGroup cols={2}>
  <Card title="Rota nativa do Hono" icon="globe" color="#C9CBD1">

  </Card>

  <Card title="Verificacao de assinatura" icon="shield-check" color="#C9CBD1">

  <Card title="Callbacks tipados" icon="code" color="#C9CBD1">

  </Card>

  <Card title="Amigavel a varios runtimes" icon="cloud" color="#C9CBD1">
    O mesmo estilo de integracao funciona bem em deployments Hono para edge e runtimes compativeis com Node.
  </Card>
</CardGroup>

## Exemplo basico

```ts
import { Hono } from 'hono';
import { Webhooks } from '@rewritetoday/hono';

const app = new Hono();

app.post(
  '/webhooks/rewrite',
  Webhooks({
    secret: process.env.REWRITE_WEBHOOK_SECRET!,
    onPayload({ id, type, data }) {
      console.log({ id, type, data });
    },
  }),
);
```

## Boas praticas

<Accordion title="Mantenha segredos na configuracao do runtime">
  Carregue o segredo de assinatura a partir de variaveis de ambiente ou bindings, sem hardcode no codigo da aplicacao.
</Accordion>

<Accordion title="Responda rapido as entregas">
  Mantenha a rota focada em verificacao e dispatch. Mova trabalho mais longo para filas ou tarefas em background.
</Accordion>

<Accordion title="Deduplique retries">
  A Rewrite pode reenviar webhooks, entao persista o id do evento e torne efeitos colaterais idempotentes.
</Accordion>

<Accordion title="Callbacks de evento disponiveis">
  O adapter suporta `onSMSOTP`, `onMessageSent`, `onMessageBatch`, `onMessageQueued`, `onMessageDelivered`, `onMessageScheduled`, `onMessageFailed`, `onMessageCanceled` e o hook generico `onPayload`.
</Accordion>
