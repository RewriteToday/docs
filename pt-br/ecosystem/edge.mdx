[`@rewritetoday/edge`](https://www.npmjs.com/package/@rewritetoday/edge) e o adapter da Rewrite para runtimes edge que expoem as APIs padrao de `Request` e `Response`.

Repositorio: [github.com/RewriteToday/adapters](https://github.com/RewriteToday/adapters)

## Quando usar

- Quando voce roda em workers ou edge functions com modelo padrao da Web API.
- Quando quer um handler de webhook orientado ao runtime, sem dependencia de framework.
- Quando precisa de verificacao de assinatura e callbacks tipados na fronteira da rota.
- Quando quer um ponto de integracao pequeno e adequado para ambientes com budget de execucao apertado.

## Capacidades principais

<CardGroup cols={2}>
  <Card title="Orientado ao runtime" icon="cloud" color="#C9CBD1">

## Exemplo basico

```ts
import { Webhooks } from '@rewritetoday/edge';

export default {
  fetch: Webhooks({
    secret: 'whsec_...',
    onPayload({ id, type, data }) {
      console.log({ id, type, data });
    },
  }),
};
```

## Boas praticas

<Accordion title="Leia o segredo a partir de bindings ou env">
  Evite hardcode do segredo do webhook. Injete esse valor por bindings do worker, config de edge ou variaveis de ambiente.
</Accordion>

<Accordion title="Retorne rapido e descarregue trabalho pesado">
  Endpoints de webhook devem confirmar a entrega rapidamente. Mova trabalho caro para filas, durable objects ou jobs em background.
</Accordion>

<Accordion title="Deduplique pelo id do evento">
  Persista o id do evento e mantenha o processamento idempotente para que retries nao gerem efeitos duplicados.
</Accordion>

<Accordion title="Callbacks de evento disponiveis">
  O adapter suporta `onSMSOTP`, `onMessageSent`, `onMessageBatch`, `onMessageQueued`, `onMessageDelivered`, `onMessageScheduled`, `onMessageFailed`, `onMessageCanceled` e o hook generico `onPayload`.
</Accordion>
