`@rewritetoday/zod` fornece schemas oficiais em Zod para a API da Rewrite, permitindo validar requests, responses, recursos reutilizaveis e payloads de webhook em runtime.

Repositorio: [github.com/RewriteToday/zod](https://github.com/RewriteToday/zod)

## Quando usar

- Quando voce quer validacao em runtime alem dos tipos TypeScript.
- Quando recebe input externo antes de chamar endpoints da Rewrite.
- Quando precisa validar payloads de webhook antes de processa-los.
- Quando quer schemas versionados alinhados aos modelos da API Rewrite.

## Exemplo de validacao de request

```ts
import { RESTPostCreateTemplateBody } from '@rewritetoday/zod/v1';

const data = RESTPostCreateTemplateBody.parse({
  name: 'order_shipped',
  content: 'Hi {{first_name}}, your order {{order_id}} is on the way.',
  variables: [
    { name: 'first_name', fallback: 'customer' },
    { name: 'order_id', fallback: '0000' },
  ],
});

console.log(data);
```

## Exemplo de validacao de webhook

```ts
import { WebhookEvent } from '@rewritetoday/zod/v1';

const result = WebhookEvent.safeParse(payload);

if (!result.success) {
  console.error({ error: result.error.flatten() });
  throw new Error('Payload de webhook Rewrite invalido');
}

console.log({ data: result.data });
```

## Boas praticas

<Accordion title="Valide nas bordas">
  Faca parse do input externo antes de ele chegar na logica de negocio para falhar cedo e com previsibilidade.
</Accordion>

<Accordion title="Mantenha versoes explicitas">
  Importe de um namespace especifico, como `@rewritetoday/zod/v1`, para manter upgrades intencionais.
</Accordion>

<Accordion title="Combine validacao e transporte">
  Use `@rewritetoday/zod` junto com `@rewritetoday/rest` e `@rewritetoday/types` para fechar validacao, rotas e transporte na mesma stack.
</Accordion>
