# Visao geral

A Rewrite suporta a chave `i18n` tanto para mensagens diretas quanto para templates.

Use `message` como fallback padrao e informe variacoes localizadas em `i18n`.
A Rewrite resolve a melhor variante com base no numero de destino (`to`) inferindo pais/regiao no formato E.164.

## Quando usar i18n

- Voce envia o mesmo fluxo para varios paises.
- Voce quer uma unica integracao de API com copy por idioma.
- Voce precisa de fallback seguro quando nao houver match de locale.

## Formato do payload

```json
{
  "to": "+5511999999999",
  "message": "Your OTP code is 123456",
  "i18n": {
    "pt-BR": "Seu codigo OTP e 123456",
    "es": "Tu codigo OTP es 123456",
    "en": "Your OTP code is 123456"
  }
}
```

## Ordem de resolucao

1. A Rewrite inspeciona o prefixo do numero de destino (`to`) e infere locale/pais.
2. Se existir match exato em `i18n` (exemplo: `pt-BR`), usa esse texto.
3. Se nao houver locale exato, pode usar a chave de idioma base quando disponivel (exemplo: `pt`).
4. Se nao houver mapeamento, usa `message`.

<Info>
Sempre envie `message` completo como fallback. Nao dependa apenas das chaves de `i18n`.
</Info>

## Formatos recomendados de chave i18n

- Idioma: `en`, `pt`, `es`
- Idioma + regiao: `en-US`, `pt-BR`, `es-MX`

Use chaves no padrao BCP-47 para matching previsivel.

## Templates + i18n

Ao usar templates, `i18n` segue o mesmo comportamento:

- Mantenha um template/mensagem padrao.
- Adicione variantes localizadas em `i18n`.
- Mantenha variaveis/placeholders consistentes em todos os idiomas.

## Boas praticas

- Mantenha placeholders identicos em todos os idiomas (exemplo: `{{code}}`).
- Respeite limites regionais de tamanho de SMS.
- Prefira locales explicitos (`pt-BR`) quando o texto muda por pais.
- Monitore entrega por locale para detectar mapeamentos ausentes.
- Teste com formatos reais dos seus principais paises.

## Erros comuns

<Tabs sync={false}>
  <Tab title="Sem message de fallback">
    Enviar apenas chaves em `i18n` sem `message` padrao pode quebrar envios para destinos sem match.
  </Tab>
  <Tab title="Placeholders inconsistentes">
    Se um locale usar placeholders diferentes, a interpolacao pode falhar em runtime.
  </Tab>
  <Tab title="Chave de locale fora do padrao">
    Evite chaves customizadas como `portugues-brasil`. Use BCP-47.
  </Tab>
</Tabs>
