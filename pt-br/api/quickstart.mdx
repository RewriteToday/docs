## Antes de comecar

- Crie uma API key da Rewrite no seu dashboard.
- Mantenha sua API key somente no backend.
- Use numeros de destino no formato **E.164** (exemplo: `+5511999999999`).

## Envie seu primeiro SMS

<Steps>
  <Step title="Monte sua requisicao" titleSize="h3">
    Use esta requisicao cURL base como ponto de partida.

    <SmsRequestPlayground />
  </Step>

  <Step title="Confira a resposta" titleSize="h3">
    Execute esta requisicao e compare sua saida com o exemplo abaixo:

    ```bash
    curl -X POST "https://api.rewritetoday.com/v1/messages" \
      -H "Authorization: Bearer rw_..." \
      -H "Content-Type: application/json" \
      -d '{
        "to": "+5511999999999",
        "message": "Seu codigo OTP e 123456",
        "metadata": { "source": "quickstart" }
      }'
    ```

    Uma requisicao bem-sucedida retorna um objeto de SMS criado semelhante a:

    ```json
    {
      "id": "733501234567890123",
      "to": "+5511999999999",
      "message": "Seu codigo OTP e 123456",
      "status": "QUEUED"
    }
    ```
  </Step>
</Steps>

<Info>
Use seu proprio numero verificado/de teste em desenvolvimento. Nao exponha API keys e segredos em codigo cliente.

## Solucao de problemas

<CardGroup cols={2}>
  <Card title="401 Unauthorized" icon="shield-alert" color="#C9CBD1">
    Sua API key esta ausente ou invalida. Verifique o token bearer no header `Authorization` do cURL.
  </Card>

  <Card title="422 Unprocessable Entity" icon="triangle-alert" color="#C9CBD1">
    Seu payload esta invalido. Confira o formato do numero em `to` e os campos obrigatorios.
  </Card>
</CardGroup>
