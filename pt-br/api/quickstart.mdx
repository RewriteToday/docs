# Antes de comecar

- Crie uma API key no dashboard da Rewrite.
- Mantenha a chave apenas no backend.
- Use um numero valido em E.164, por exemplo `+5511999999999`.
- Se quiser comportamento deterministico de testes, use [Sandbox](/pt-br/sandbox/introduction).

## Envie seu primeiro SMS

```bash
curl -X POST "https://api.rewritetoday.com/v1/messages" \
  -H "Authorization: Bearer rw_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: msg:quickstart:2026-05-06T18:30" \
  -d '{
    "to": "+5511999999999",
    "content": "Rewrite: seu codigo de verificacao e 123456",
    "tags": {
      "flow": "quickstart"
    }
  }'
```

Resposta de sucesso:

```json
{
  "ok": true,
  "data": {
    "id": "748395130237498500",
    "createdAt": "2026-05-06T18:30:00.000Z",
    "analysis": {
      "characters": 41,
      "encoding": "gsm7",
      "segments": {
        "concat": 153,
        "count": 1,
        "reason": "fits",
        "single": 160
      }
    },
    "sandbox": false
  }
}
```

## Proximos passos comuns

- Consulte a mensagem depois com `GET /messages/{id}`.
- Envie lotes com `POST /messages/batch`.
- Adicione [webhooks](/pt-br/webhooks/introduction) para receber [`message.sent`](/pt-br/webhooks/events/message-sent), [`message.delivered`](/pt-br/webhooks/events/message-delivered), [`message.failed`](/pt-br/webhooks/events/message-failed), [`message.received`](/pt-br/webhooks/events/message-received), [`message.batch`](/pt-br/webhooks/events/message-batch) e [`sms.otp`](/pt-br/webhooks/events/sms-otp).
- Use [templates](/pt-br/api/i18n) quando precisar ler variantes localizadas e enviar por `templateId`.
- Use [Sandbox magic numbers](/pt-br/sandbox/magic-numbers) para forcar resultados deterministas em testes.

## Troubleshooting

- `400 INVALID_JSON_BODY`: body, telefone ou validacao especifica da rota falhou.
- `401 INVALID_OR_MISSING_API_KEY`: bearer token ausente ou invalido.
- `403 API_KEY_MODE_MISMATCH`: o modo da API key nao corresponde ao modo atual do projeto.
- `403 PROJECT_SUBSCRIPTION_REQUIRED` ou `PROJECT_BILLING_PAYMENT_REQUIRED`: projeto bloqueado para envio.
- `429 RATE_LIMITED`: faca backoff e respeite os headers de retry.
