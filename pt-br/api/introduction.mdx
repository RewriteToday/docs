# Visao geral

A Rewrite API v1 expoe endpoints publicos de projeto para enviar SMS, verificar OTP, gerenciar templates e webhooks e consultar logs de entrega.

## Base URL

Todos os endpoints publicos ficam em:

```text
https://api.rewritetoday.com/v1
```

## Autenticacao

Rotas publicas usam sua API key:

```http
Authorization: Bearer rw_...
```

## Convencoes de request

- Use `Content-Type: application/json` em requests que nao sao `GET`.
- Os fluxos publicos de envio e OTP aceitam hoje apenas numeros brasileiros com prefixo `+55`.
- Criacao de mensagem inline usa `content`.
- Envios por template usam `templateId` com `variables`.
- Rotas de create/send aceitam `Idempotency-Key`.

Exemplo de request publica:

```bash
curl -X POST "https://api.rewritetoday.com/v1/messages" \
  -H "Authorization: Bearer rw_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: msg:login-otp:2026-03-19T18:30" \
  -d '{
    "to": "+5511999999999",
    "content": "Rewrite: seu codigo de verificacao e 123456"
  }'
```

## Convencoes de resposta

Respostas bem-sucedidas usam o envelope `ok` / `data`:

```json
{
  "ok": true,
  "data": {
    "id": "748395130237498500",
    "createdAt": "2026-03-19T18:30:00.000Z",
    "analysis": {
      "characters": 39,
      "encoding": "gsm7",
      "segments": {
        "concat": 153,
        "count": 1,
        "reason": "fits",
        "single": 160
      }
    }
  }
}
```

Rotas de lista paginadas retornam `cursor` no topo.
