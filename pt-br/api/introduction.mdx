## Base URL

Use este host para todos os endpoints da API:

```text
https://api.rewritetoday.com
```

Exemplo de endpoint para criar mensagem:

```text
https://api.rewritetoday.com/v1/messages
```

<Info>
Use apenas HTTPS. Nunca envie API keys por canais inseguros.
</Info>

## Versionamento

A Rewrite versiona endpoints no caminho da URL.

Versao documentada atual:

- `v1`

Regras de versionamento:

- Atualizacoes backward-compatible podem sair dentro da mesma versao.
- Mudancas breaking saem em uma nova rota de versao.
- Mantenha a versao explicita em todos os endpoints hardcoded.

## Autenticacao

Autentique todas as requisicoes com uma API key secreta do seu dashboard Rewrite.

Header esperado:

```http
Authorization: Bearer rw_...
```

Exemplo de requisicao:

```bash
curl -X POST "https://api.rewritetoday.com/v1/messages" \
  -H "Authorization: Bearer rw_..." \
  -H "Content-Type: application/json" \
  -d '{
    "to": "+15551234567",
    "message": "Your OTP code is 123456"
  }'
```

<Warning>
Nao exponha API keys secretas em codigo cliente web/mobile. Mantenha-as no backend.
</Warning>

## Convencoes da API

- `Content-Type`: use `application/json` para payloads JSON.
- Formato de telefone: use E.164 (exemplo: `+15551234567`).
- Erros: respostas nao-2xx retornam payloads estruturados de erro.
- Limites: alta pressao de requests pode retornar `429` com headers de retry.
