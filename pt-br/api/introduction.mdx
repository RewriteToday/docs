# Visao geral

A Rewrite API v1 expoe endpoints publicos de projeto para enviar SMS, verificar OTP, gerenciar templates e webhooks, e consultar logs de request e entregas de webhook.

## Base URL

Todos os endpoints publicos ficam em:

```text
https://api.rewritetoday.com/v1
```

## Autenticacao

Rotas publicas usam sua API key:

```http
Authorization: Bearer rw_...
```

## Convencoes de request

- Use `Content-Type: application/json` em requests que nao sao `GET`.
- Os fluxos publicos de envio e OTP aceitam um numero E.164 direto em `to` ou um contato salvo do projeto em `contact`.
- Criacao de mensagem inline usa `content`.
- Envios por template usam `templateId` com `variables`.
- Metadata de mensagens e templates usa objeto chave/valor, nao array de pares.
- Rotas de create/send aceitam `Idempotency-Key`.

Exemplo de request:

```bash
curl -X POST "https://api.rewritetoday.com/v1/messages" \
  -H "Authorization: Bearer rw_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: msg:login-otp:2026-05-06T18:30" \
  -d '{
    "to": "+5511999999999",
    "content": "Rewrite: seu codigo de verificacao e 123456",
    "tags": {
      "flow": "login",
      "origin": "checkout"
    }
  }'
```

## Convencoes de resposta

Respostas bem-sucedidas usam o envelope `ok` / `data`:

```json
{
  "ok": true,
  "data": {
    "id": "748395130237498500",
    "createdAt": "2026-05-06T18:30:00.000Z",
    "analysis": {
      "characters": 41,
      "encoding": "gsm7",
      "segments": {
        "concat": 153,
        "count": 1,
        "reason": "fits",
        "single": 160
      }
    },
    "sandbox": false
  }
}
```

Rotas de lista paginadas retornam `cursor` no topo.

## Modelo publico de eventos

- O ciclo de entrega agora inclui `message.delivered`.
- Trafego inbound aparece como [`message.received`](/pt-br/webhooks/events/message-received).
- Entregas de webhook ficam na secao `Deliveries`, incluindo `GET /deliveries` e `GET /webhooks/{id}/deliveries`.
