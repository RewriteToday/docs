## O que sao os SDKs da Rewrite?

Os **SDKs da Rewrite** (Software Development Kits) sao bibliotecas oficiais que simplificam a comunicacao com a API da Rewrite.  
Eles oferecem interfaces ergonomicas por linguagem para voce integrar SMS transacionais no seu aplicativo com rapidez e seguranca, sem lidar com detalhes de HTTP de baixo nivel.

Os SDKs foram desenhados para parecer naturais em cada ecossistema e permanecer alinhados com a API.

<Card title="Por que usar os SDKs da Rewrite?" horizontal>
  * **Integracao simplificada**: Metodos prontos para os principais endpoints da API
  * **Tipagem forte**: Suporte de tipos quando disponivel
  * **Erros consistentes**: Modelo de erro unificado entre linguagens
  * **Curva de aprendizado menor**: Sem precisar dominar todos os detalhes da API no inicio
  * **Sempre atualizados**: Evoluem junto com novos recursos da API
</Card>

## Perguntas frequentes

<Accordion title="Como instalar o SDK da minha linguagem?">
  Cada SDK inclui instrucoes de instalacao em seu repositorio no GitHub. Na maioria dos casos, voce pode instalar com o gerenciador de pacotes da linguagem, como npm, pip, go modules, Composer ou Bundler.
</Accordion>

<Accordion title="Os SDKs funcionam em desenvolvimento e producao?">
  Sim. Todos os SDKs funcionam em ambos os ambientes.
</Accordion>

<Accordion title="O que fazer se eu encontrar um bug em um SDK?">
  Abra uma issue no repositorio do SDK afetado no GitHub. Monitoramos e corrigimos regularmente.
</Accordion>

<Accordion title="Nao existe SDK para minha linguagem. E agora?">
  Conte para a gente qual linguagem voce precisa. Contribuicoes da comunidade sao bem-vindas.
</Accordion>
