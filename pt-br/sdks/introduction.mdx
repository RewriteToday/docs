## O que são os SDKs do Rewrite?

Os **SDKs do Rewrite** (Software Development Kits) são bibliotecas oficiais que simplificam a comunicação com a API do Rewrite.  
Eles oferecem interfaces ergonômicas e específicas para cada linguagem, permitindo que você integre SMS transacionais ao seu aplicativo de forma rápida e segura — sem lidar com detalhes de baixo nível de HTTP.

Os SDKs são projetados para parecer naturais em cada ecossistema, mantendo total alinhamento com a API subjacente.

<Card title="Por que usar os SDKs do Rewrite?" horizontal>
  * **Integração simplificada**: Métodos prontos para todos os endpoints principais da API
  * **Tipagem forte**: Suporte completo a tipos quando disponível
  * **Tratamento de erros consistente**: Modelo de erros unificado entre linguagens
  * **Menor curva de aprendizado**: Não é necessário conhecer todos os detalhes da API desde o início
  * **Sempre atualizados**: Os SDKs evoluem junto com as novas funcionalidades da API
</Card>

## Perguntas frequentes

<Accordion title="Como instalar o SDK da minha linguagem?">
  Cada SDK inclui instruções de instalação em seu respectivo repositório no GitHub. Na maioria dos casos, você pode instalá-lo usando o gerenciador de pacotes da sua linguagem, como npm, pip, go modules, Composer ou Bundler.
</Accordion>

<Accordion title="Os SDKs funcionam tanto em desenvolvimento quanto em produção?">
  Sim. Todos os SDKs funcionam em ambos os ambientes. Sem mágica, sem boilerplate.
</Accordion>

<Accordion title="O que devo fazer se encontrar um bug em um SDK?">
  Abra uma issue no repositório do GitHub do SDK afetado. Monitoramos ativamente os problemas e lançamos correções com frequência.
</Accordion>

<Accordion title="Não existe um SDK para a linguagem que eu uso. O que fazer?">
  Conte para a gente qual linguagem você precisa. Contribuições da comunidade são bem-vindas, e ficaremos felizes em apoiar novos SDKs oficiais sempre que possível.
</Accordion>
