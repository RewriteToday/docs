# Visao geral

A Rewrite envia webhooks como requests HTTPS `POST` com body JSON.

As familias publicas de evento hoje sao:

- [`sms.otp`](/pt-br/webhooks/events/sms-otp)
- [`message.sent`](/pt-br/webhooks/events/message-sent)
- [`message.batch`](/pt-br/webhooks/events/message-batch)
- [`message.queued`](/pt-br/webhooks/events/message-queued)
- [`message.failed`](/pt-br/webhooks/events/message-failed)
- [`message.canceled`](/pt-br/webhooks/events/message-canceled)
- [`message.scheduled`](/pt-br/webhooks/events/message-scheduled)

[`message.delivered`](/pt-br/webhooks/events/message-delivered) esta em WIP.

## Registro do webhook

Voce pode criar webhooks no dashboard ou via `POST /webhooks`.

Exemplo:

```bash
curl -X POST "https://api.rewritetoday.com/v1/webhooks" \
  -H "Authorization: Bearer rw_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Eventos de entrega",
    "endpoint": "https://example.com/webhooks/rewrite",
    "events": ["message.sent", "message.failed", "sms.otp"]
  }'
```

## Formato da entrega

A Rewrite envia hoje:

- `Content-Type: application/json`
- `User-Agent: rewrite-webhooks/1.0`
- `svix-id`
- `svix-timestamp`
- `svix-signature`

Exemplo de payload:

```json
{
  "id": "748395130237498900",
  "createdAt": "2026-03-19T18:42:11.000Z",
  "type": "message.sent",
  "data": {
    "id": "748395130237498500",
    "projectId": "748395130237498412",
    "to": "+5511999999999",
    "tags": [
      { "name": "use_case", "value": "otp_login" }
    ],
    "type": "SMS",
    "status": "SENT",
    "content": "Rewrite: seu codigo de verificacao e 123456",
    "country": "br",
    "analysis": {
      "characters": 43,
      "encoding": "gsm7",
      "segments": {
        "concat": 153,
        "count": 1,
        "reason": "fits",
        "single": 160
      }
    },
    "error": null,
    "deliveredAt": null,
    "scheduledAt": null,
    "templateId": null
  }
}
```

## Validacao de assinatura

Valide a assinatura antes de confiar no body.

Detalhes importantes:

- Use o raw body da request.
- Leia `svix-id`, `svix-timestamp` e `svix-signature` nos headers.
- Verifique contra o segredo retornado por `POST /webhooks` ou `GET /webhooks/{id}`.

Para o fluxo completo de verificacao, a string canonica assinada e um exemplo em Node, veja [Assinatura](/pt-br/webhooks/signature).

## Boas praticas do handler

- Retorne qualquer `2xx` rapidamente.
- Processe trabalho pesado de forma assincrona.
- Deduplique por id do evento.
- Persista o payload bruto para debugging e replay.
