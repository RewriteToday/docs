---
title: "message.sent"
description: "Mensagem aceita para entrega no provider."
---

`message.sent` e emitido quando a Rewrite recebe a confirmacao do provider para o dispatch da mensagem.

```json
{
  "id": "748395130237498911",
  "createdAt": "2026-03-19T18:42:13.000Z",
  "type": "message.sent",
  "sandbox": false,
  "data": {
    "id": "748395130237498500",
    "projectId": "748395130237498412",
    "contact": "Fernanda",
    "contactId": "748395130237498515",
    "to": "+5511999999999",
    "tags": { "flow": "otp_login", "origin": "checkout" },
    "sandbox": false,
    "type": "SMS",
    "status": "SENT",
    "content": "Rewrite: seu codigo de verificacao e 123456",
    "country": "br",
    "analysis": {
      "characters": 43,
      "encoding": "gsm7",
      "segments": { "concat": 153, "count": 1, "reason": "fits", "single": 160 }
    },
    "error": null,
    "deliveredAt": null,
    "scheduledAt": null,
    "templateId": null
  }
}
```

Notas:

- `data.status` e `SENT`.
- `data.error` e `null`.
- Envios imediatos normalmente emitem [`message.queued`](./message-queued) antes deste evento.
- Envios agendados podem emitir `message.sent` depois preservando o `data.scheduledAt` original.
