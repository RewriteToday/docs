## Base URL

Usa este host para todos los endpoints de API:

```text
https://api.rewritetoday.com
```

Ejemplo de endpoint para crear mensaje:

```text
https://api.rewritetoday.com/v1/messages
```

<Info>
Usa solo HTTPS. No envies API keys por canales inseguros.
</Info>

## Versionado

Rewrite versiona endpoints en la ruta URL.

Version documentada actual:

- `v1`

Reglas de versionado:

- Cambios backward-compatible pueden salir dentro de la misma version.
- Cambios breaking se publican en una nueva ruta de version.
- Mantén la version explicita en todos los endpoints hardcodeados.

## Autenticacion

Autentica cada solicitud con una API key secreta de tu dashboard de Rewrite.

Header esperado:

```http
Authorization: Bearer rw_...
```

Ejemplo de solicitud:

```bash
curl -X POST "https://api.rewritetoday.com/v1/messages" \
  -H "Authorization: Bearer rw_..." \
  -H "Content-Type: application/json" \
  -d '{
    "to": "+15551234567",
    "message": "Your OTP code is 123456"
  }'
```

<Warning>
Nunca expongas API keys secretas en codigo de cliente web/mobile. Guardalas del lado servidor.
</Warning>

## Convenciones de API

- `Content-Type`: usa `application/json` para payloads JSON.
- Formato telefonico: usa E.164 (ejemplo: `+15551234567`).
- Errores: respuestas no-2xx devuelven payloads de error estructurados.
- Limites: alta presion de requests puede devolver `429` con headers de retry.
