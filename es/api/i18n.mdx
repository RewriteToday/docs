# Resumen

Rewrite admite una clave `i18n` tanto para mensajes directos como para plantillas.

Usa `message` como fallback predeterminado y añade variantes localizadas en `i18n`.
Rewrite resuelve la mejor variante segun el pais/la region inferidos desde el numero de destino (`to`) en formato E.164.

## Cuando usar i18n

- Envias el mismo flujo a varios paises.
- Quieres una sola integracion API con copy especifico por locale.
- Necesitas un comportamiento de fallback seguro cuando no exista coincidencia de locale.

## Forma del payload

```json
{
  "to": "+5511999999999",
  "message": "Tu codigo OTP es 123456",
  "i18n": {
    "pt-BR": "Seu codigo OTP e 123456",
    "es": "Tu codigo OTP es 123456",
    "en": "Your OTP code is 123456"
  }
}
```

## Orden de resolucion

1. Rewrite inspecciona el prefijo del numero de destino (`to`) e infiere locale/pais.
2. Si existe coincidencia exacta de locale en `i18n` (por ejemplo `pt-BR`), Rewrite la usa.
3. Si no existe locale exacto, Rewrite puede usar una clave a nivel idioma cuando exista (por ejemplo `pt`).
4. Si no se encuentra mapeo, Rewrite usa `message`.

<Info>
Envia siempre un fallback `message` completo. No dependas solo de claves `i18n`.
</Info>

## Formatos de clave i18n soportados

- Idioma: `en`, `pt`, `es`
- Idioma + region: `en-US`, `pt-BR`, `es-MX`

Se recomiendan claves de estilo BCP-47 para coincidencias predecibles.

## Plantillas + i18n

Al usar plantillas, `i18n` sigue el mismo comportamiento:

- Mantén una plantilla/mensaje predeterminado.
- Añade variantes especificas por locale en `i18n`.
- Mantén variables/placeholders consistentes en todos los locales.

## Buenas practicas

- Mantén placeholders identicos entre locales (ejemplo: `{{code}}`).
- Mantén la longitud del mensaje dentro de los limites regionales de operador.
- Prefiere locales explicitos (`pt-BR`) cuando tono/redaccion cambien por pais.
- Añade monitorizacion por locale para detectar mapeos faltantes.
- Prueba con formatos de telefono reales de tus principales paises.

## Errores comunes

<Tabs sync={false}>
  <Tab title="Sin mensaje fallback">
    Enviar solo claves `i18n` sin un `message` por defecto puede romper la entrega en destinos sin coincidencia.
  </Tab>
  <Tab title="Placeholders inconsistentes">
    Si un locale usa nombres de placeholder diferentes, la interpolacion en runtime puede fallar.
  </Tab>
  <Tab title="Formato de locale incorrecto">
    Evita claves personalizadas como `brazilian-portuguese`. Usa claves de estilo BCP-47.
  </Tab>
</Tabs>
