`@rewritejs/types` centraliza contratos de la API de Rewrite para mantener consistencia entre servicios y paquetes.

Repositorio: [github.com/RewriteToday/types](https://github.com/RewriteToday/types)

## Que incluye

## Ejemplo con builders de rutas

```ts
import { Routes } from '@rewritejs/types';

const listRoute = Routes.apiKeys.list('748395130237498368');
const createRoute = Routes.apiKeys.create('748395130237498368');

console.log({ listRoute, createRoute });
```

## Ejemplo con enums

```ts
import { APIKeyScope, WebhookEventType } from '@rewritejs/types';

const scopes = [APIKeyScope.ReadTemplates, APIKeyScope.ReadWebhooks];
const events = [WebhookEventType.SMSDelivered, WebhookEventType.SMSFailed];

console.log({ scopes, events });
```

## Por que usarlo incluso sin REST?

- Evita strings sueltos en codigo de negocio.
- Mejora seguridad de refactor en codebases grandes.
- Hace visible el drift de rutas/contratos durante code review.
- Estandariza el uso de la API entre equipos y servicios.

<Info>
Usar `@rewritejs/types` junto con `@rewritejs/rest` mantiene rutas, enums y payloads alineados de extremo a extremo.
</Info>
