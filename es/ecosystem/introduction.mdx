El ecosistema Rewrite es un conjunto de paquetes enfocados que funcionan en conjunto.
Puedes usarlos por separado o combinarlos para tener una integracion tipada y resiliente.

## Arquitectura del ecosistema

<CardGroup cols={3}>
  <Card title="Contratos" icon="code" color="#C9CBD1">
    `@rewritejs/types` ofrece enums, builders de rutas y contratos de payload compartidos.
  </Card>

  <Card title="Transporte" icon="route" color="#C9CBD1">
    `@rewritejs/rest` entrega control HTTP de bajo nivel con helpers de rutas de Rewrite.
  </Card>

  <Card title="Capa de producto" icon="package" href="/es/sdks/introduction" color="#C9CBD1">

## Que paquete debo usar?

<Accordion title="Quiero integrar rapido y con menos boilerplate">
  Empieza con los SDKs oficiales. Ofrecen APIs de mayor nivel y suelen ser la via mas rapida para equipos de producto.
</Accordion>

<Accordion title="Necesito control total de HTTP/runtime">
  Usa `@rewritejs/rest`, opcionalmente junto con `@rewritejs/types` para tipar rutas y payloads.
</Accordion>

<Accordion title="Solo necesito contratos y builders de rutas">
  Usa solamente `@rewritejs/types` cuando ya tienes tu propio cliente HTTP o capa de abstraccion.
</Accordion>
