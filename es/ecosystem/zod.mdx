`@rewritejs/zod` ofrece schemas oficiales en Zod para la API de Rewrite, de modo que puedas validar requests, responses, recursos reutilizables y payloads de webhook en runtime.

Repositorio: [github.com/RewriteToday/zod](https://github.com/RewriteToday/zod)

## Cuando usarlo

- Cuando quieres validacion en runtime ademas de tipos TypeScript.
- Cuando recibes input externo antes de llamar endpoints de Rewrite.
- Cuando necesitas validar payloads de webhook antes de procesarlos.
- Cuando quieres schemas versionados alineados con los modelos de la API de Rewrite.

## Ejemplo de validacion de request

```ts
import { RESTPostCreateTemplateBody } from '@rewritejs/zod/v1';

const data = RESTPostCreateTemplateBody.parse({
  name: 'order_shipped',
  content: 'Hi {{first_name}}, your order {{order_id}} is on the way.',
  variables: [
    { name: 'first_name', fallback: 'customer' },
    { name: 'order_id', fallback: '0000' },
  ],
});

console.log(data);
```

## Ejemplo de validacion de webhook

```ts
import { WebhookEvent } from '@rewritejs/zod/v1';

const result = WebhookEvent.safeParse(payload);

if (!result.success) {
  console.error({ error: result.error.flatten() });
  throw new Error('Payload de webhook Rewrite invalido');
}

console.log({ data: result.data });
```

## Buenas practicas

<Accordion title="Valida en los bordes">
  Haz parse del input externo antes de que llegue a tu logica de negocio para fallar rapido y con previsibilidad.
</Accordion>

<Accordion title="Manten versiones explicitas">
  Importa desde un namespace especifico como `@rewritejs/zod/v1` para que los upgrades sean deliberados.
</Accordion>

<Accordion title="Combina validacion y transporte">
  Usa `@rewritejs/zod` junto con `@rewritejs/rest` y `@rewritejs/types` para cubrir validacion, rutas y transporte en una sola stack.
</Accordion>
