## ¿Qué son los SDKs de Rewrite?

Los **SDKs de Rewrite** (Software Development Kits) son librerías oficiales que simplifican la comunicación con la Rewrite API.  
Ofrecen interfaces ergonómicas y específicas por lenguaje para que puedas integrar SMS transaccionales en tu aplicación de forma rápida y segura, sin tratar con detalles HTTP de bajo nivel.

Los SDKs están diseñados para sentirse naturales en cada ecosistema mientras se mantienen totalmente alineados con la API subyacente.

<Card title="¿Por qué usar los SDKs de Rewrite?" horizontal>
  * **Integración simplificada**: Métodos preconstruidos para todos los endpoints principales de la API
  * **Tipado fuerte**: Soporte de tipos de primera clase donde esté disponible
  * **Manejo de errores consistente**: Modelos de error unificados entre lenguajes
  * **Curva de aprendizaje menor**: No necesitas entender cada detalle de la API desde el inicio
  * **Sincronizados**: Los SDKs se actualizan junto con nuevas funcionalidades de la API
</Card>

## Preguntas frecuentes

<Accordion title="¿Cómo instalo el SDK para mi lenguaje?">
  Cada SDK incluye instrucciones de instalación en su repositorio de GitHub. En la mayoría de los casos, puedes instalarlo usando el gestor de paquetes de tu lenguaje, como npm, pip, go modules, Composer o Bundler.
</Accordion>

<Accordion title="¿Los SDKs funcionan tanto en desarrollo como en producción?">
  Sí. Todos los SDKs funcionan en ambos entornos. Sin magia, sin boilerplate.
</Accordion>

<Accordion title="¿Qué debo hacer si encuentro un bug en un SDK?">
  Abre un issue en el repositorio de GitHub del SDK afectado. Monitoreamos activamente los issues y publicamos correcciones con regularidad.
</Accordion>

<Accordion title="No hay SDK para mi lenguaje. ¿Qué debo hacer?">
  Cuéntanos qué lenguaje necesitas. Las contribuciones de la comunidad son bienvenidas y estaremos encantados de apoyar nuevos SDKs oficiales cuando sea posible.
</Accordion>
